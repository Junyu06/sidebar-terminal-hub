# Sidebar Terminal Hub

Sidebar Terminal Hub 是一个把可交互终端直接嵌入 VS Code 辅助侧边栏的扩展。

它的核心工作流是：

**打开侧边栏终端 → 把常用命令会话固定在侧边 → 不切到底部面板也能持续运行 CLI 工具**

这个项目的重点是把侧边栏做成一个真正实用的终端工作区，而不是默认继续叠加和定位无关的终端功能。

[English README](./README.md)

---

## 解决的问题

VS Code 自带终端本身很好用，但大多数情况下它都放在底部面板。

当你希望下面这些体验时，底部布局并不总是合适：

- 一边写代码一边持续看到终端输出
- 把长时间运行的 CLI 会话固定在侧边
- 把侧边栏当成一个轻量命令中心来用

Sidebar Terminal Hub 的目标就是减少这类切换成本，把可交互终端放到本来就适合工作流停留的侧边栏里。

---

## 功能特性

- 在辅助侧边栏中直接显示可交互终端
- 支持多个终端会话标签
- 支持直接键盘输入、粘贴、运行命令
- 支持 ANSI / TUI 类终端输出
- 支持状态栏按钮快速打开侧边栏终端
- 支持在视图标题栏中创建和关闭会话
- 支持自定义快捷命令按钮，适配你自己的工作流

---

## 典型使用场景

- 一边编码一边持续查看终端输出
- 把 Codex、测试命令、构建脚本或 dev server 固定在侧边
- 同时保留多个 CLI 会话，而不用频繁切换到底部面板
- 把辅助侧边栏变成一个轻量终端 hub

---

## 设计方向

这个仓库当前作为一个独立维护的 fork 使用。

目标不是机械地跟随上游的每一个终端交互细节更新，  
而是围绕 **sidebar terminal hub** 这个方向，保持更聚焦的产品边界。

---

## 开发

```bash
npm install
npm run compile
```

然后在 VS Code 中按 `F5` 启动扩展开发宿主窗口。

---

## 打包

```bash
npm.cmd run package:patch
```

这个命令会自动：

- 补丁版本号 `+1`
- 编译扩展
- 生成新的 `.vsix`
